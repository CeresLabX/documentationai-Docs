## Choose a project

CeresLab includes three small apps with distinct user-facing workflows: shared media rooms, trivia games, and a stock-market sandbox. Use this page as the starting point for the current documented behavior.

## What is documented

The pages in this initial set document behavior verified from the connected application code. They focus on what users and API consumers can see or call, not on internal implementation details.

<Callout kind="info">

These docs intentionally cover only verified current behavior. If a feature is not represented in the reviewed code evidence, it is not described here.


</Callout>

## Current coverage

- **Mod Room** — Room-code sharing, first-item queue auto-start, host playback controls, tracker playback, and Koofr browsing.
- **Trivia** — Public question endpoints, optional difficulty filtering, response fields, random-question empty-state behavior, and fixed game category buckets.
- **Stocks Sandbox** — Simulator endpoints, chart windows, refresh behavior, watchlist deletion rules, encoded tickers, historical prices, buys, and portfolio valuation fields.
